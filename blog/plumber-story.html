The Translation: Plumbing IS Computing

Every concept in AI agent engineering has a direct plumbing equivalent. I didn't need a CS degree. I needed to recognize the patterns I already knew:

The Plumber's Glossary for AI

  • Pipes = data flow between agents (stdin/stdout, bash pipes)
  • Wrenches = tools (grep, jq, sed, curl — each one purpose-built)
  • Valves = approval gates (operator must authorize before elevated commands execute)
  • Leaks = prompt injection (untrusted input escaping into the control plane)
  • Clogs = context overflow, OOM errors (too much data for the pipe diameter)
  • Branching pipes = model router (directing flow to the right destination)
  • Pressure testing = audit trails (jsonl logs verifying every joint holds)
  • Backflow preventer = input validation (regex checks before data touches the system)
  • Main shutoff = kill switch (one command stops the entire fleet)

Twenty years of fixing physical systems taught me the one skill that matters in software: understanding flow. Where data comes from, where it goes, what happens when it gets stuck, and how to route it efficiently.

The Bash-Pipe Method

The core innovation isn't AI. It's not using AI.

Most AI platforms route everything through the language model. Every question, every lookup, every file operation — it all hits the LLM, and you pay per token. That's like running hot water through every pipe in the house when you only need to fill one sink.

The Bash-Pipe Method flips this:

User request → "scan example.com for vulnerabilities"
  |
  ├─ [bash] Validate input (regex: is it a domain?)     → $0
  ├─ [bash] Run dig, curl, whois in parallel              → $0
  ├─ [bash] Parse results with grep/jq/sed                → $0
  ├─ [bash] Filter to relevant findings (30 lines)        → $0
  |
  └─ [LLM] Synthesize findings into assessment            → 1 API call
                                                           (local = $0)

Result: 90% of the work never touches a model. The LLM only does what bash can't — synthesize, judge, and generate natural language. Everything else is pre-processed for free.

The Method IS the Product

The "Script Factory" or "Script Keeper" method:

  1. Pipe — Pre-process everything with bash/grep/jq/sed before it touches an LLM.
  2. Script — Write bash megascripts that generate entire codebases via heredocs. One script = one deliverable.
  3. Delegate — The AI writes the script. Bash executes it. No back-and-forth.
  4. Conserve — Free APIs for 80% of work. Paid only for synthesis.
  5. Save — Every script is reusable, sellable, documentable proof of concept.

A solo builder with no CS degree, on an old laptop, with $20 of API credits, built in 5 days what would cost a funded startup $100K+ and 8-16 months with a team of 6-8 engineers.
