The Script Keeper Method

How a plumber gets 3.5x more work out of every AI session — and why you're probably burning 70% of your tokens on nothing.

I run 15 AI agents on $0/month in API costs. I built a SaaS platform, a 3D-printed hardware line, and a full marketing pipeline — all on a basic Claude Code subscription.

The secret isn't a better model. It's not a better prompt. It's discipline.

I call it the Script Keeper Method. Here's how it works.

The Problem: You're Wasting 70% of Your AI Budget

Watch how most people use AI tools:

  • They read the same file 3-5 times because they forgot what was in it
  • They search the web for "best practices" instead of just building
  • They get stuck in debug loops — retrying the same failed command 10 times
  • They narrate what they're about to do instead of doing it
  • They explore the codebase aimlessly instead of targeting exactly what they need
  • They run one task at a time when five could run in parallel

Every one of those habits burns tokens. Tokens cost money. Or if you're on a subscription, they eat your context window — which means less work per session.

Every multi-step operation gets scripted, saved, and reused. We don't waste tokens. We pipe and execute.

The 8 Rules

1. Read Once, Edit Once

Never re-read a file you already have in context. If you just read it, edit it now. Don't come back to it later.

2. No Exploratory Searches

If you know the file path, read it directly. Don't grep for something you already know the location of. Every search costs tokens.

3. Parallel Everything

If you need to read 4 files, read them all at once — not one at a time. If you need to run 3 independent tasks, run them in parallel. Time is money. Tokens are money.

4. No Debug Loops

If something doesn't work after 2 attempts — scrap it. Build an alternative. Don't burn 10 rounds retrying the same failed approach. That's not persistence, it's waste.

5. Script Everything

If you do something more than once, script it. Save it. Every multi-step operation becomes a .sh or .py file in your scripts directory. Next time, you run the script instead of explaining the task again.

6. Local Tools Are Free

Your terminal, your shell, your grep, your jq — those cost nothing. Pre-filter data locally before sending it to the AI. Send the smallest possible input.

7. No Fluff

Don't narrate what you're about to do. Don't explain your reasoning unless asked. Don't add comments to code that doesn't need them. Say what you did, not what you're thinking about doing.

8. Pipe and Execute

Chain operations together. Send precise instructions. Get the job done in the fewest possible calls.

The Pipe Patterns

Here's how Script Keepers use AI in practice:

# One-shot code review — single API call, zero conversation overhead
git diff | claude -p "Review: security, bugs, performance. Suggest fixes."

# Log analysis — pre-filter locally, send only what matters
grep -i 'error\|exception' app.log | sort | uniq -c | sort -nr | head -30 \
  | claude -p "Group by root cause, suggest fixes."

# Quick refactor
cat src/utils/helpers.ts | claude -p "3 improvements for readability."

# Config comparison
diff -u old.yaml new.yaml | claude -p "Explain changes. Flag breaking diffs."

# JSON analysis
jq '.[] | select(.level == "error")' events.json \
  | claude -p "Common patterns, root causes, mitigations."

Notice the pattern: local tools do the heavy lifting. The AI gets the smallest possible input and a precise instruction. One call. One result. Done.
